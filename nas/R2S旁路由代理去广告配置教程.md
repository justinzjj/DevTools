# R2S 旁路由代理与去广告配置教程

本文记录 NanoPi R2S / R2S 作为家庭网络旁路由的完整配置流程，目标是：

- 主路由继续负责拨号、Wi-Fi 和 DHCP。
- R2S 作为旁路由，负责 Clash 代理和 DNS 去广告。
- 先让单台设备测试，稳定后再决定是否扩大到更多设备。

当前家庭网络参数：

```text
主路由地址：192.168.31.1
主路由 DHCP：开启
DHCP 地址池：192.168.31.5 - 192.168.31.254
子网掩码：255.255.255.0
推荐 R2S 地址：192.168.31.2
```

最终推荐结构：

```text
光猫 / 入户
   |
主路由 192.168.31.1
   |
交换机 / Wi-Fi / 局域网
   |
R2S 旁路由 192.168.31.2
```

## 1. 主路由检查

在主路由后台确认以下配置：

```text
局域网 IP：192.168.31.1
子网掩码：255.255.255.0
DHCP 服务：开启
DHCP 起始 IP：192.168.31.5
DHCP 结束 IP：192.168.31.254
```

主路由的 `DNS1`、`DNS2`、`默认网关` 先保持空白或默认，不要一开始就改成 R2S。

因为 DHCP 地址池从 `192.168.31.5` 开始，所以 `192.168.31.2` 不会被主路由自动分配给其他设备，适合作为 R2S 的固定地址。

等 R2S 接入网络后，建议在主路由的 DHCP 静态 IP 分配里绑定：

```text
设备名：R2S
IP 地址：192.168.31.2
```

## 2. 选择系统

推荐使用：

```text
iStoreOS + OpenClash + AdGuard Home
```

原因：

- iStoreOS 基于 OpenWrt，适合家用软路由。
- 插件中心更友好，安装 OpenClash 和 AdGuard Home 比原版 OpenWrt 省心。
- 中文资料多，适合先把旁路由跑通。

如果以后想更深入学习 OpenWrt，可以再准备第二张 TF 卡刷原版 OpenWrt。

## 3. 刷入 iStoreOS

准备材料：

```text
R2S
16GB TF 卡
读卡器
电脑
balenaEtcher 或 Raspberry Pi Imager
iStoreOS for R2S 镜像
```

下载 R2S 对应的 iStoreOS 镜像，文件名通常包含：

```text
r2s
squashfs
img.gz
```

不要下载 `r4s`、`r5s`、`x86` 等其他设备镜像。

使用 balenaEtcher 写入 TF 卡：

```text
Flash from file -> 选择 iStoreOS 的 .img.gz 镜像
Select target -> 选择 TF 卡
Flash -> 等待写入和校验完成
```

刷完后，如果电脑提示磁盘无法读取或要求格式化，不要格式化，直接安全弹出 TF 卡。

## 4. 第一次启动 R2S

第一次启动建议先不要接入家庭主路由。

接线：

```text
电脑 ---- 网线 ---- R2S LAN 口
R2S 插电
```

等待 1-3 分钟后，浏览器访问：

```text
http://192.168.100.1
```

iStoreOS 默认密码通常是：

```text
password
```

如果打不开，可以把电脑临时设置为：

```text
IP 地址：192.168.100.10
子网掩码：255.255.255.0
网关：192.168.100.1
```

然后再次访问：

```text
http://192.168.100.1
```

进入后台后，先修改管理员密码，并设置时区为 `Asia/Shanghai`。

## 5. 配置 R2S 为旁路由

进入：

```text
网络 -> 接口 -> LAN -> 修改
```

设置：

```text
IPv4 地址：192.168.31.2
IPv4 子网掩码：255.255.255.0
IPv4 网关：192.168.31.1
DNS 服务器：192.168.31.1
```

关闭 R2S 的 DHCP：

```text
网络 -> 接口 -> LAN -> DHCP 服务器
勾选：忽略此接口
```

保存并应用。

之后 R2S 后台地址会变成：

```text
http://192.168.31.2
```

## 6. 接入家庭网络

改成正常接线：

```text
主路由 LAN 口 / 交换机 ---- R2S LAN 口
```

电脑连回家里的 Wi-Fi 或有线网络，然后访问：

```text
http://192.168.31.2
```

如果能打开，说明 R2S 已经作为局域网设备接入成功。

进入：

```text
系统 -> 软件包
```

尝试更新软件包列表。如果能更新，说明 R2S 自身联网正常。

## 7. 安装和配置 OpenClash

在 iStoreOS 软件中心安装：

```text
OpenClash
```

安装后进入：

```text
服务 -> OpenClash
```

先安装或更新内核，优先选择：

```text
Mihomo / Clash Meta 内核
```

导入订阅：

```text
服务 -> OpenClash -> 配置订阅
```

添加：

```text
名称：自定义
订阅地址：你的 Clash / Mihomo 订阅链接
```

更新配置后，在配置文件管理中选择刚下载的配置。

基础设置建议：

```text
代理模式：规则模式
运行模式：Fake-IP 或 TUN
开机启动：开启
绕过大陆 IP：开启
绕过局域网地址：开启
DNS 服务：开启
```

第一次建议先用默认推荐模式。如果遇到部分 App 异常，再尝试在 Fake-IP、TUN、Redir-Host 之间切换。

启动 OpenClash 后，确认状态为运行中。

## 8. 先测试代理

不要马上让全家设备都走 R2S。先拿一台设备测试，例如 Mac 或手机。

把测试设备网络设置为：

```text
IP 地址：自动或手动固定
子网掩码：255.255.255.0
网关 / 路由器：192.168.31.2
DNS：192.168.31.2
```

测试：

```text
国内网站是否正常
国外网站是否正常
代理出口是否正确
App 是否正常加载
```

如果代理正常，再继续配置去广告。

## 9. 安装 AdGuard Home

在 iStoreOS 软件中心安装：

```text
AdGuard Home
```

安装后访问管理界面，常见地址类似：

```text
http://192.168.31.2:3000
```

初始化时建议：

```text
管理界面监听：0.0.0.0:3000
DNS 服务监听：0.0.0.0:53
```

如果提示 53 端口被占用，通常是系统自带 dnsmasq 占用了端口。

推荐处理方式：

```text
网络 -> DHCP/DNS -> 高级设置
DNS 服务器端口：54
```

把系统 dnsmasq 改到 `54` 端口后，让 AdGuard Home 使用 `53` 端口。

## 10. 配置 AdGuard Home 上游 DNS

AdGuard Home 里进入：

```text
设置 -> DNS 设置 -> 上游 DNS 服务器
```

推荐上游 DNS：

```text
127.0.0.1:7874
```

这通常是 OpenClash 的 DNS 监听地址。

目标链路是：

```text
客户端 DNS 请求
-> AdGuard Home 过滤广告
-> OpenClash DNS 分流
-> 直连或代理
```

如果 `127.0.0.1:7874` 暂时不通，可以先使用公共 DNS 测试：

```text
223.5.5.5
119.29.29.29
1.1.1.1
8.8.8.8
```

但最终更推荐交给 OpenClash DNS，国外域名分流会更顺。

## 11. 配置去广告规则

AdGuard Home 进入：

```text
过滤器 -> DNS 封锁清单
```

可以添加：

```text
https://anti-ad.net/easylist.txt
https://adrules.top/adrules_domainset.txt
https://raw.githubusercontent.com/privacy-protection-tools/anti-AD/master/anti-ad-domains.txt
```

建议先添加 1-2 个规则，稳定后再增加。规则太多可能造成误杀，例如 App 图片加载失败、登录异常、视频软件异常。

## 12. 最终推荐配置

主路由：

```text
IP：192.168.31.1
DHCP：开启
DHCP 范围：192.168.31.5 - 192.168.31.254
默认网关：保持默认或留空
DNS1/DNS2：保持默认或留空
```

R2S：

```text
IP：192.168.31.2
子网掩码：255.255.255.0
网关：192.168.31.1
DNS：192.168.31.1
DHCP：关闭
```

OpenClash：

```text
开机启动：开启
代理模式：规则模式
运行模式：Fake-IP / TUN / Redir-Host 按实际情况选择
DNS 服务：开启
绕过大陆 IP：开启
绕过局域网地址：开启
```

AdGuard Home：

```text
DNS 监听：0.0.0.0:53
管理界面：0.0.0.0:3000
上游 DNS：127.0.0.1:7874
```

需要代理和去广告的设备：

```text
网关：192.168.31.2
DNS：192.168.31.2
```

普通设备：

```text
网关：192.168.31.1
DNS：自动
```

## 13. 是否要全局切换

推荐长期使用方式：

```text
只让需要代理和去广告的设备使用 R2S
```

也就是在这些设备上手动设置：

```text
网关：192.168.31.2
DNS：192.168.31.2
```

如果希望全家设备都走 R2S，可以在主路由 DHCP 设置里填写：

```text
默认网关：192.168.31.2
DNS1：192.168.31.2
```

但这会让 R2S 成为全家网络关键节点。R2S 或 OpenClash 出问题时，全家设备可能无法正常上网，所以建议等稳定后再考虑。

## 14. 常见故障

### R2S 后台打不开

检查：

```text
R2S 是否是 192.168.31.2
电脑是否在 192.168.31.0/24 网段
是否接在 R2S LAN 口
192.168.31.2 是否被其他设备占用
```

可以在电脑上测试：

```bash
ping 192.168.31.2
```

### 设置网关后设备不能上网

检查：

```text
R2S 网关是否是 192.168.31.1
R2S DNS 是否是 192.168.31.1
R2S 是否能更新软件包
OpenClash 是否启动
客户端网关是否是 192.168.31.2
客户端 DNS 是否是 192.168.31.2
```

### 代理正常但广告没过滤

检查：

```text
客户端 DNS 是否是 192.168.31.2
AdGuard Home 是否监听 53 端口
AdGuard Home 查询日志里是否有客户端请求
过滤规则是否启用
```

### 广告过滤正常但代理异常

检查：

```text
OpenClash 是否运行中
订阅是否更新成功
节点是否可用
配置文件是否选中
OpenClash DNS 是否可用
```

### 网页打不开但 ping IP 能通

大概率是 DNS 链路问题。重点检查：

```text
客户端 DNS -> 192.168.31.2
AdGuard Home -> 53 端口
AdGuard Home 上游 -> 127.0.0.1:7874
OpenClash DNS -> 正常运行
```

### 全家断网的快速回滚

如果改过主路由 DHCP 默认网关或 DNS，立即改回：

```text
默认网关：192.168.31.1 或留空
DNS：自动或留空
```

如果只改了单台设备，直接把该设备网络改回自动获取，或手动改成：

```text
网关：192.168.31.1
DNS：自动
```

必要时直接拔掉 R2S，不应影响主路由正常上网。

## 15. 推荐操作顺序

第一次配置时按这个顺序：

```text
1. 刷 iStoreOS 到 TF 卡
2. 首次启动 R2S
3. 修改 R2S LAN 为 192.168.31.2
4. 关闭 R2S DHCP
5. 接入主路由
6. 确认 R2S 能联网
7. 安装 OpenClash
8. 导入订阅并测试代理
9. 只让一台设备使用 R2S 网关和 DNS
10. 安装 AdGuard Home
11. 配置 AdGuard Home 监听 53 端口
12. 配置 AdGuard Home 上游到 OpenClash DNS
13. 添加少量去广告规则
14. 稳定后再扩大到更多设备
```

## 16. 配置完成后备份

在 iStoreOS 后台进入：

```text
系统 -> 备份/升级 -> 生成备份
```

保存备份文件。

同时记录：

```text
R2S IP：192.168.31.2
主路由 IP：192.168.31.1
OpenClash 运行模式
AdGuard Home 上游 DNS
使用 R2S 的设备列表
```

这样以后重刷系统或换 TF 卡时，可以快速恢复。
