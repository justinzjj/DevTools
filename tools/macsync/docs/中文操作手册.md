# macsync 中文操作手册

## 1. macsync 是什么

`macsync` 是一个基于 Git 的多设备文件同步管理工具，目标是服务你的三设备工作流：

```text
MacBook / Mac mini / local server
```

推荐架构：

```text
                  GitHub
                    ^
                    |
                  origin
                    |
MacBook  <---->  macsync  <---->  Mac mini
                    |
              local server / Gitea
```

其中：

- `local server` 上运行 Gitea，作为内网里的“本地 GitHub”。
- MacBook 和 Mac mini 是工作节点。
- 每个仓库保留两个 remote：
  - `origin`：GitHub，不受 macsync 影响。
  - `macsync`：内网 Gitea，用于设备间同步。

`macsync` 不替代 Git。它只是把你常用的同步动作封装成更顺手的命令。

## 2. 适合同步什么

适合：

- 代码仓库
- Markdown 笔记
- Obsidian 中的文本笔记
- 项目文档
- 论文草稿
- 配置文件
- 小脚本

不建议直接同步：

- `node_modules/`
- `.venv/`
- Docker volume
- 链节点数据
- 数据库目录
- 大型实验输出
- 大量 PDF / PPT / Word 附件
- 模型文件

这些大文件或运行态文件可以后续用 Git LFS、git-annex、rsync 或文件服务器处理。

## 3. 安装位置

当前安装脚本使用用户态安装，不需要 `sudo`。

默认位置：

```text
~/.local/share/macsync
~/.local/bin/macsync
~/.local/bin/macsync-server
```

如果命令找不到，先把下面这一行加入 `~/.zshrc` 或当前 shell：

```bash
export PATH="$HOME/.local/bin:$PATH"
```

验证安装：

```bash
macsync version
macsync-server compose --http-port 3000 --ssh-port 2222 --data-dir /tmp/macsync-gitea-test
```

## 4. local server 端部署

### 4.1 安装 macsync-server

在服务器上执行：

```bash
./scripts/install-server.sh /path/to/macsync
```

如果是从 MacBook 复制到服务器安装，可以先把项目传到 `/tmp`，再安装：

```bash
rsync -az /path/to/macsync/ codex-linux-wg:/tmp/macsync-src/
ssh codex-linux-wg
cd /tmp/macsync-src
./scripts/install-server.sh .
```

### 4.2 生成 Gitea Docker Compose

在服务器上：

```bash
macsync-server compose --http-port 3000 --ssh-port 2222 --data-dir /srv/gitea > docker-compose.yml
```

检查内容：

```bash
cat docker-compose.yml
```

启动 Gitea：

```bash
docker compose up -d
```

打开浏览器：

```text
http://<server-ip>:3000
```

例如你的 SSH 配置里 server 是 `codex-linux-wg`，可以先尝试：

```text
http://codex-linux-wg:3000
```

### 4.3 Gitea 初始化建议

第一次打开 Gitea 时，建议：

- 创建管理员账号：`justin`
- SSH 端口使用 compose 中映射的端口，例如 `2222`
- 数据目录使用 `/srv/gitea`
- 先创建几个测试仓库，再迁移真实仓库

建议先建：

```text
justin/DevTools
justin/CrossDataGen
justin/TrustMap-doc
```

## 5. Mac 节点配置

### 5.1 安装 macsync

在 MacBook 或 Mac mini 上执行：

```bash
./scripts/install-mac-node.sh /path/to/macsync
```

验证：

```bash
macsync version
```

### 5.2 配置文件位置

默认配置文件：

```text
~/.config/macsync/config.yml
```

可以参考项目里的示例：

```text
examples/macsync.yml
```

示例：

```yaml
sync_remote: macsync
github_remote: origin
gitea_host: codex-linux-wg
gitea_web_url: http://codex-linux-wg:3000
gitea_ssh_user: git
gitea_owner: justin
repos:
  - name: DevTools
    path: ~/ICT/4-code/0-DevTools/DevTools
    branch: main
    gitea_repo: DevTools
    github: git@github.com:justinzjj/DevTools.git

  - name: TrustMap-doc
    path: ~/ICT/1-个人知识库/4-论文工作/1-TrustMap
    branch: main
    gitea_repo: TrustMap-doc
    github: git@github.com:justinzjj/TrustMap-doc.git
```

字段说明：

- `sync_remote`：内网同步 remote 名称，建议固定为 `macsync`。
- `github_remote`：GitHub remote 名称，通常是 `origin`。
- `gitea_host`：Gitea SSH host，例如 `codex-linux-wg`。
- `gitea_web_url`：Gitea 网页地址。
- `gitea_ssh_user`：Gitea SSH 用户，一般是 `git`。
- `gitea_owner`：Gitea 仓库拥有者或组织名。
- `repos[].path`：本地路径。
- `repos[].branch`：同步分支。
- `repos[].gitea_repo`：Gitea 上的仓库名。
- `repos[].github`：GitHub 地址，仅用于记录，不会自动覆盖 `origin`。

### 5.3 查看同步映射

```bash
macsync list
```

输出会显示：

```text
仓库名
Gitea SSH remote
Gitea Web URL
本地路径
分支
```

## 6. 首次接入已有仓库

假设本地已有仓库：

```text
~/ICT/4-code/0-DevTools/DevTools
```

并且 Gitea 上已经创建：

```text
justin/DevTools
```

在 Mac 上运行：

```bash
macsync setup-remotes --dry-run
```

确认输出没问题后：

```bash
macsync setup-remotes
```

它会添加：

```bash
git remote add macsync git@codex-linux-wg:justin/DevTools.git
```

不会删除或修改 `origin`。

首次推送：

```bash
git push macsync main
```

如果你的分支不是 `main`，按配置中的 `branch` 为准。

## 7. 日常使用

### 7.1 开始工作前同步

在 MacBook 或 Mac mini 上：

```bash
macsync sync
```

它相当于对每个配置仓库执行：

```bash
git fetch macsync
git pull --ff-only macsync <branch>
```

这里使用 `--ff-only`，是为了避免自动合并造成意外。

### 7.2 离开设备前交接

```bash
macsync handoff
```

如果仓库有未提交改动，它会执行：

```bash
git add -A
git commit -m "wip: handoff from <hostname>"
git push macsync <branch>
```

如果仓库没有未提交改动，它会直接：

```bash
git push macsync <branch>
```

### 7.3 同步到 GitHub

```bash
macsync backup
```

它会依次执行：

```bash
git push macsync <branch>
git push origin <branch>
```

这适合在你想把内网中心和 GitHub 都更新时使用。

### 7.4 查看状态

```bash
macsync status
```

它会显示：

```text
仓库名
当前分支
未提交改动数量
相对 macsync 的 ahead/behind
相对 origin 的 ahead/behind
```

## 8. 冲突处理

`macsync` 不发明新的冲突系统。冲突基本就是 Git 冲突。

例如：

```bash
macsync sync
```

如果失败，常见原因是：

- 本地有未提交改动。
- 本地和 Gitea 都有新提交，无法 fast-forward。
- 同一个文件在两台 Mac 上都被改了。
- 远程分支不存在。
- SSH 连不上 Gitea。

处理方式仍然是 Git 的方式。

### 8.1 本地有未提交改动

先看状态：

```bash
git status
```

选择提交：

```bash
git add -A
git commit -m "wip: local changes"
macsync sync
```

或者临时藏起来：

```bash
git stash
macsync sync
git stash pop
```

### 8.2 无法 fast-forward

`macsync sync` 使用 `git pull --ff-only`。如果两边都有提交，会失败。

先看分叉情况：

```bash
git fetch macsync
git log --oneline --graph --decorate --all -20
```

如果你想正常合并：

```bash
git merge macsync/<branch>
```

解决冲突后：

```bash
git add -A
git commit
macsync handoff
```

如果你想 rebase：

```bash
git rebase macsync/<branch>
```

解决冲突后：

```bash
git add -A
git rebase --continue
macsync handoff
```

### 8.3 SSH 连接失败

测试：

```bash
ssh git@codex-linux-wg
```

或根据 Gitea SSH 端口：

```bash
ssh -p 2222 git@codex-linux-wg
```

检查：

- Gitea 是否启动。
- SSH key 是否加入 Gitea。
- `gitea_host` 是否正确。
- SSH 端口是否正确。
- `~/.ssh/config` 是否需要配置别名。

### 8.4 remote 已存在

如果 `macsync setup-remotes` 提示 remote 已存在：

```bash
git remote -v
```

如果地址正确，不需要处理。

如果地址错误：

```bash
git remote set-url macsync git@codex-linux-wg:justin/DevTools.git
```

### 8.5 分支不存在

先看远程分支：

```bash
git ls-remote --heads macsync
```

首次推送：

```bash
git push -u macsync <branch>
```

## 9. 问题反馈是否和 Git 一致

基本一致。

`macsync` 的核心动作都是 Git 命令：

```bash
git fetch
git pull --ff-only
git add
git commit
git push
git remote add
```

所以大多数错误来自 Git 本身，例如：

```text
non-fast-forward
Your local changes would be overwritten
Permission denied (publickey)
remote rejected
repository not found
merge conflict
```

这也是设计目标：不要创造一套新的同步语义，而是复用 Git 的版本管理、冲突检测和错误提示。

`macsync` 负责做三件事：

- 根据配置找到哪些仓库要同步。
- 把常用 Git 操作串成安全命令。
- 保留 `origin`，额外使用 `macsync` remote 做内网同步。

## 10. 安全测试方式

任何新版本上线前，先跑隔离测试：

```bash
PATH="$HOME/.local/bin:$PATH" ./scripts/e2e-temp-git-sync.sh
```

这个测试只会使用 `/tmp` 下的临时目录：

```text
/tmp/macsync-e2e.*/server/demo.git
/tmp/macsync-e2e.*/node-a/demo
/tmp/macsync-e2e.*/node-b/demo
```

不会碰真实 `~/ICT`，不会碰 `/srv/git`，不会碰 `/srv/gitea`。

## 11. 推荐使用习惯

到一台设备开始工作：

```bash
macsync status
macsync sync
```

离开这台设备：

```bash
macsync handoff
```

一天结束或需要外部备份：

```bash
macsync backup
```

遇到问题：

```bash
git status
git remote -v
git log --oneline --graph --decorate --all -20
```

然后按 Git 的方式解决。
